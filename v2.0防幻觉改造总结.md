# 规则梳理工具 · v2.0 防幻觉改造总结

> 完成日期：2026-06-17
> 改造范围：8 个根因（A-H）全修，P0-P3 一次性完成
> 验证状态：23 项单元测试全部通过

---

## 一、改造背景

用户反馈三大问题：虚构规则、原文摘录不精准、规则梳理不细致。经根因剖析定位 8 个根因（A-H），其中 A（无语义校验）和 B（摘录=整块文本）是同一结构性缺口的两侧——只要系统不要求模型输出精准摘录、不校验文字是否来自原文，文字级幻觉就能穿透全部五道门禁。

## 二、改造内容（按根因）

### P0 根因 B：摘录精准化（结构性缺口修复）

| 改动项 | 文件 | 说明 |
|--------|------|------|
| 6 个 prompt 新增 source_excerpt 字段 | P1/P1_law/P2/P3/P4/P5 | OUTPUT FORMAT 要求模型输出 30-150 字逐字摘录；忠实度铁律升级为"四铁律" |
| take_excerpt 共用函数 | harness.py | 模型给了用模型摘录，未给回退整块 |
| 5 个管道改用 take_excerpt | p1/p2/p3/p4/p5 | 赋值 excerpt_fallback + raw_block_text |
| verify_excerpt 后处理校验 | fidelity.py | 归一化子串复核 + difflib 模糊回填(≥0.8 标 fuzzy) + <0.8 回退标 mismatch |
| RuleCandidate 新增字段 | parsers.py | excerpt_fallback, excerpt_mismatch, excerpt_fuzzy, raw_block_text |

### P0 根因 A：语义忠实度（文字幻觉拦截）

| 改动项 | 文件 | 说明 |
|--------|------|------|

| 接入校验链 | orchestrator.py | _apply_fidelity_gate 中 check_fidelity 后调用 |
| 接入综合置信度 | confidence.py | 新增 semantic 权重 0.15 |

## 三、配置变更

config.default.yaml 权重从 5 项调整为 6 项：

```yaml
weights:
  self: 0.15        # 0.25 → 0.15
  consistency: 0.25
  struct: 0.15
  conflict: 0.05
  fidelity: 0.25    # 0.30 → 0.25
  semantic: 0.15    # 新增
```

## 四、校验链流程（v2.0）

```
管道层                          校验链（_apply_fidelity_gate）
──────                          ──────────────────────────
take_excerpt                    verify_excerpt
  ↓ 模型摘录或回退整块             ↓ 归一化子串复核 + difflib 模糊回填
RuleCandidate                   check_fidelity
  ↓                               ↓ 数值 token grounding（摘录范围内）
genericize_instances            check_voice_match
  ↓ 重写后重新 take_excerpt        ↓ soft→strong + neutral→strong(禁止性)
_finalize                       check_semantic_fidelity
  ↓                               ↓ 法律术语+主体名词回溯
_apply_fidelity_gate            ──────────────────────
  ↓                             output_target 决策：
evaluate_confidence               semantic偏离>0.7 → discarded
  ↓ 六重门加权                     fidelity失败≥2 → discarded
build_rule_ids                    占位规则 → placeholder
                                  其他 → main
```
